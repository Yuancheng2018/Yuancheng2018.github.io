Http&Https

HTTP优缺：

优点
  1. http的灵活性高，可扩展性强，从http1.0到http1.1再到http2.x，http协议一直在进行扩展新的属性。
  2. 可靠传输，因为http协议是基于tcp协议的一种应用层协议，tcp协议就是可靠传输协议。
  3. 请求应答，有来有回
  4. 无状态的，每一个请求都是相互独立的，默认不需要保存上下文的信息，组集群可让负载均衡设备把请求任意分配。
缺点
  1. 明文传输不安全
  2. 复用一个TCP连接，会发生对头阻塞情况， 1.1引入长连接，在一个tcp上发送多个http，两种模式：管道化&非管道化

a) 非管道化：完全串行，请求>响应>请求>响应

b) 管道化：允许在收到响应之前发送请求，但是由于http协议是无序的，服务 器端必须串行，依然存在阻塞

HTTP2中引入帧/消息/数据流的概念（区别于四层模型），每个请求/响应被称为消 息，每个消息都被拆分成若干个帧进行传输，每个帧都分配一个序号。每个帧在传 输是属于一个数据流，而一个连接上可以存在多个流，各个帧在流和连接上独立传 输，到达之后在组装成消息，这样就避免了请求/响应阻塞。但是http还是基于tcp 实现，存在tcp队头阻塞（tcp可靠传输，体现就是按序到达。）

HTTP3 QUIC 的形式实现一个全新的传输层协议。它运行在不可靠的 UDP 协议之 上。但它包括 TCP 的所有特性（可靠性、拥塞控制、流量控制、排序等），且集成 了TLS，不允许未加密的连接

  1. 无状态的，头比身子大
  2. http1.1客户端拉取资源，要啥给啥，HTTP2可以提前把可能用到的JS等推给客户端

HTTPS：

HTTPS是一种通过计算机网络进行安全通信的传输协议，经由HTTP进行通信，利用SSL/TLS建立全信道，加密数据包。HTTPS使用的主要目的是提供对网站服务器的身份认证，同时保护交换数据的隐私与完整性。TLS是传输层加密协议，前身是SSL协议，相当于SSL3.1。
